Как установить Ghost локально


Ускоренная локальная установка для запуска Ghost на вашем компьютере или настройка для разработки темы

Обзор
Локальный запуск Ghost - это самый простой способ запустить свою собственную копию программного обеспечения и иметь возможность выполнять с ним некоторые локальные разработки. К концу этого руководства вы завершите локальную установку Ghost, которая выполняется в режиме разработки с использованием SQLite3.

Эта установка не подходит для производственного использования или внесения вклада в ядро.

Предпосылки
Чтобы установить Ghost локально, вам потребуется следующее:

Компьютер под управлением macOS, Windows или Linux
Поддерживаемая версия Node.js
Либо yarn, либо npm для управления пакетами
Чистый, пустой каталог на вашем компьютере


Установка Ghost-CLI
Ghost-CLI - это инструмент командной строки, который поможет вам быстро и легко установить и настроить Ghost для использования. Модуль npm может быть установлен с помощью npm или yarn на локальном компьютере, в зависимости от ваших предпочтений.

npm install ghost-cli@latest -g
После установки вы всегда можете запустить ghost help, чтобы просмотреть список доступных команд.

Установить Призрак
В вашем терминале вставьте компакт-диск в пустой каталог и запустите команду установки:

призрак устанавливает локальную
Как только установка будет завершена, вы сможете получить доступ к своему новому сайту на http://localhost:2368 и http://localhost:2368/ghost чтобы получить доступ к администратору Ghost ✨

Вот и все! С тобой покончено.

Ваша публикация настроена в режиме разработки с меньшим кэшированием
База данных SQLite3 настраивается автоматически и находится в /content/data/
Журналы отправляются только в stdout


Запуск и остановка
Ghost запускается в отдельном фоновом процессе и остается запущенным до тех пор, пока вы не остановите его или не перезагрузите компьютер. Таким образом, вы можете найти эти команды полезными для укрощения его:

ghost stop для остановки Ghost
ghost start для запуска Ghost
ghost log просмотры журналов
ghost ls, чтобы перечислить все запущенные блоги Ghost
Запустите справку ghost для получения списка доступных команд или ознакомьтесь с полной документацией Ghost-CLI.

Диагностика
Для устранения неполадок и устранения ошибок попробуйте выполнить поиск в этой документации и разделе часто задаваемых вопросов, чтобы найти информацию о распространенных сообщениях об ошибках.

Разработка Тем
Чтобы работать с темой Ghost Handlebars локально, ваша пользовательская тема всегда должна быть размещена в каталоге верхнего уровня /content/themes/.

Живая перезагрузка
Все изменения, внесенные в файлы темы Ghost, будут автоматически перезагружены. Если вы добавите какие-либо новые файлы в свою тему во время разработки, вам нужно будет перезапустить Ghost, чтобы изменения вступили в силу.

Проверка с помощью GScan
GScan - это инструмент, который проверяет темы Ghost на совместимость с последними версиями Ghost. Ghost автоматически запускает этот инструмент при загрузке или активации темы. В целях разработки вы также можете выполнить эти проверки самостоятельно, установив его локально.

# Установить gscan глобально
npm install gscan -g

# Сканируйте каталог тем на предмет совместимости
gscan /путь/к/призраку/контенту/темам/касперу

# Сканируйте zip-файл темы на совместимость
gscan -z /path/to/downloads/theme.zip
GScan также можно получить по адресу gscan.ghost.org , где вы можете подписаться на последние обновления в качестве разработчика темы Ghost.

Что дальше
Вы завершили локальную установку Ghost — поздравляю! Теперь вы можете ознакомить Ghost с его возможностями и посмотреть, что это такое, или сразу приступить к разработке пользовательской темы Ghost.

Когда вы будете готовы запустить свой сайт в производство, следуйте одному из этих руководств.

Для получения дополнительной информации о разработке темы ознакомьтесь с документацией по теме Handlebars и ознакомьтесь с учебными пособиями.
